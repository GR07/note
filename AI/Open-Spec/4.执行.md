# 第一步

/opsx:explore 描述...

和模型聊天帮你思考细化想要的功能



# 第二步

确认没问题后，执行 /opsx:propose

此条命令一口气生成四份文档


- proposal.md 

介绍需求背景，需要的功能。

并且会根据功能点详细的在 specs 目录下生成一份规格说明（WHEN: THEN:）也就是第四份文档。


- design.md

提供技术选型，以及背后原因


- tasks.md

根据上述的要求，详细列出整个开发计划。

以及可以追踪开发进度（每完成一项会打标记x）



# 第三步

四份文档生成完成后，可以调整修改，直到完全符合需求。


# 第四步

都确认完成后，执行 /opsx:apply

此命令会读取 tasks.md 里面的任务清单读取相关文档，逐步开始执行开发。


# 第五步

开发完成后，确认功能没问题，这次任务就完成了。

执行 /opsx:archive

此命令会执行归档，把当前任务的文件夹剪切到 archive 目录下，并且文件夹命名加日期。


# 归档时 specs 合并策略

- 新功能 

完整复制过去

- 已有功能-新增场景

直接把新场景追加进去

- 已有功能-修改场景

对原来功能场景进行修改
