# 定义

Open Spec 框架属于 SDD 范畴


# SDD

规格驱动开发，是在入口约束。

开发前提供清晰的需求描述，要求细致到每一个功能的规格。

本质为先确认规格文档再开发称为 SDD

相比需求不明确直接开干来回折腾，反而会节约 Token

即使懒得看生成的文档，也要这么做，因为 AI 会看，所以又节约了 Token


# TDD

测试驱动开发，在出口处管控。

要求开发成果通过严格的测试。

先写测试用例再编码称为 TDD

SDD TDD 二者可以结合（Open Spec + Super Power）


# Open Spec

## 规格文档写得好

一条命令生成默认文档体系

## 灵活度高

可以自己设计文档体系，改配置即可，无需写代码

## 文档管理得好

两条路线各司其职

- specs/ 当前最新的文档

- archive/ 历史变更轨迹



# 对开发者的要求

精准的提出需求，个人逻辑能力和表达能力
