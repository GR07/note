# 初始化

执行 openspec schema fork spec-driven my-schema

将默认的 schema 复制到当前项目 openspec 目录下


# 修改 schema.yaml

- name 

schema 名称，改成自己的名

- artifacts

文档列表，可以新增修改

- id

文档标识，命令引用

- generates

输出文件的路径，支持通配符 specs/**/*.md

- template

模板文件名，放在对应 templates/

- instruction

指令，告诉 AI 文档要写什么

- requires

文档前置依赖的文件，调整依赖关系

- apply.requires

apply 命令前必须完成的工作，最后一步

- apply.tracks

进度追踪的文件


# 修改 schema 指向

openspec/config.yaml 文件

- schema

改为 schema.yaml 里面的 name
