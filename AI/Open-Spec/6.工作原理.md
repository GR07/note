# 工作原理

Skill 只是维护基本工作流程执行，没有记录要写文档，也没记录文档要怎么写

Open Spec 则提供了一套 CLI 指令给模型执行，

模型执行对应的 CLI 指令时，

程序会从各个配置文件中组合信息（config.yaml 就是其中的一个配置文件），

动态生成文档的撰写要求。


# schema.yaml

最重要的配置文件

打开目录 C:\nvm4w\nodejs\node_modules\@fission-ai\openspec\schemas\spec-driven\schema.yaml

- artifacts 决定了需要生成哪些文件

    - id: proposal
        - template    这些文件对应的默认模板目录，也就是说可以去修改这些默认模板
        - instruction 配套撰写这些文档的指令
        - requires    说明当前文档需要依赖的文档，就是文档生成的顺序
    - id: specs
    - id: design
    - id: tasks


所以文档生成顺序是

- proposal          首先生成 需求文档

- specs / design    其次同步生成 功能规格 / 设计方案

- tasks             最后生成 开发计划



# proposal 命令的完整执行流程

执行后，将用户的自然语言发送给模型，

模型按照自然语言要求开始工作，

1. 根据需求创建文件夹

2. 程序解析 schema.yaml 把 JSON 发给模型

3. AI 就知道要生成哪些文档以及文档依赖关系

4. 程序解析 schema.yaml、config.yaml、文档模板等相关文件，整理后再把 JSON 发给模型

5. 模型得到 JSON 后，就知道创建文档时的依赖关系、模板、注意事项，AI 以此开始创建文档。

6. 上一个文档完成后，此过程不断重复，直到全部 4 个文档创建完成。


整个过程中 Skill 只负责工作流程，不与 4 个文档有绑定关系

CLI 命令负责动态生成文档撰写要求

配置文件才是定义文档的地方

所以，修改配置文件，就是修改文档体系。
