# MCP 交互流程详解


# 注册交互

首先用 JSON 配置了一个 MCP Server

Cline 使用配置中的命令去执行这个程序

执行了之后，这个程序会等待 Cline 的指令

Cline 询问当前的 MCP Server 可以提供哪些 Tool（工具函数）

MCP Server 回复并列出所有工具以及如何使用

Cline 记录回答以备后续使用

到此为止，就完成了 MCP Server 的注册，发生在开启这个 MCP Server 的一瞬间


# 提问交互

Cline 拿到用户提问传给大模型，

这里注意，和以往不一样的是，传给模型的除了用户的问题，还有所有已注册的 MCP Server 及 包含的 Tool

Cline 告诉模型，这些 Tool 可能会帮助你解决用户的问题。

模型经过思考，发现其中一个 MCP Server 的 Tool 可以回答用户的问题。

模型就让 Cline 调用一下这个 Tool ，同时提供了 Tool 的入参。

Cline 收到模型的请求后，接着命令当前这个 MCP Server 用模型返回的参数调用 Tool

MCP Server 拿到执行后的结果传给 Cline

Cline 再把结果传给模型

模型接到后，根据结果包装回答用户问题，并且把答案发给了 Cline

最终 Cline 再把结果呈现给用户。



# 总结

Cline 相当于一层代理，用户只和 Cline 交互，Cline 去和内部配置的大模型去交互。

模型不调用工具，但是模型决定调用哪个工具，根据工具的描述来决定。
