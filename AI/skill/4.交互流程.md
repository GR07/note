# 角色

用户

Claude Code

当前大模型



# 提问交互

用户输入请求

Claude Code 把用户的请求连同所有的 Skill 的名称和描述（注意这里只带了名称和描述），一起发给大模型

模型经过思考，发现用户的请求可以使用其中的一个 Skill 来解决

模型告诉 Claude Code 可以用这个 Skill

Claude Code 接到模型的相应后，会去这个 Skill 里面读取完整的 skill.md（这时才是读取完整的skill）

Claude Code 把用户请求和完整的 skill.md 发给大模型

大模型根据 skill.md 的要求生成响应，把响应发给 Claude Code

最后 Claude Code 把响应发给用户


# 核心机制

通过上述交互，可以得出 Skill 是按需加载

虽然 Skill 的名称和描述是可见的，但是只有某个 Skill 被选中后，才会把完整的 skill.md 给模型

也就是说 按需加载 节省了很多的 Token
