当你需要暂时搁置当前分支的工作，切换到其他分支进行开发时，可以使用 git stash 系列命令来管理工作现场。
以下是完整的操作流程：

1. 暂存当前分支的工作

```powershell
# 暂存所有未提交的修改（包括已跟踪文件的修改和新增文件）
git stash push -u -m "暂存当前工作：需求A开发中"

# 查看暂存列表
git stash list
```

2. 切换到其他分支进行开发

```powershell
# 切换到已存在的分支
git checkout 其他分支名

# 或者创建并切换到新分支
git checkout -b 新分支名
```

3. 在新分支完成工作后提交

```powershell
# 正常提交修改
git add .
git commit -m "完成需求B"
```

4. 切换回原分支并恢复暂存的工作

```powershell
# 切换回原分支
git checkout 原分支名

# 恢复最近一次暂存的内容（方法一：恢复后保留stash记录）
git stash apply

# 恢复最近一次暂存的内容（方法二：恢复后删除stash记录）
git stash pop

# 如果需要恢复指定的stash（根据git stash list显示的序号）
git stash apply stash@{1}
```

5. 清理不再需要的暂存记录

```powershell
# 删除最近一次暂存记录
git stash drop

# 删除所有暂存记录
git stash clear
```
